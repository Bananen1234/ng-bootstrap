Buttons

The default class for all buttons in bootstrap is .btn. It gives elements with the class base line button styles like: padding, font-weight, white-space etc. and can be applied to both buttons and anchor elements to name a few.

Bootstrap has a few different modifiers that can be used together with the base class, we primarily use .btn-primary, .btn-secondary and .btn-link, use the latter to give buttons the same behavior as normal links (used for links placed outside text segments and paragraphs as well as actions similar to "go to" or regular links). You can also use the outline version if you want an "inverted" button with border. Note that we only support .btn-primary,.btn-secondary,.btn-light,.btn-danger and .btn-dark from bootstrap, if you use any of the others they will fall back to the primary style.

Against colored backgrounds or images it's recommended to use the .btn-light or .btn-dark modifier to get good contrast .

Disabled state

Triggered by .disabled class or disabled attribute.

The .btn-light and .btn-dark classes should only be used when buttons are placed on colored or dark backgrounds/images.

Add .close to a button to get correct style and icon for a close button and don't forget to include text for screen readers with the aria label. The button will be right aligned by default. Use inside modals, alerts, badges and other components that can be closed or dismissed.

Since SEB Style is a theme for Bootstrap and Bootstrap has some additional button classes to differentiate the severity or the meaning of a button like .btn-success please note that they'll fallback to the default style.
