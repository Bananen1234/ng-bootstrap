Toast notifications

Bootstrap provides classes for toasts i.e. you can just add .toast to an element get the basic appearance of a toast, however SEB:s take on toast notifications and their design is a bit simplified and inherits the style from alerts so the bootstrap toast examples need to be altered a bit. Just add .toast together with one of the following contextual classes.

Context Usage Class name
Info Content that holds neutral information that we find useful for users and that we want to inform them about. .toast-info
Success Should be used as a response to an action or something that is considered to be positive or a success. .toast-success
Warning Not an error but important information for a user to consider or be aware of. .toast-warning
Danger Used for errors or when something goes wrong. .toast-danger
Javascript needed! Please note that the examples above are just used to illustrate the appearance of toast notifications, and the markup could be reused with any framework or toast component as long as they use the toast class. Toast notifications use the same style and appearance as alerts but you need a framework to control when and how to show them. The examples below uses a third-party library for angular called ngx-toastr, if you're using angular, please take a look at @sebgroup/ngx-toastr for more info on installation and usage.

These toasts will disappear after a set time.
